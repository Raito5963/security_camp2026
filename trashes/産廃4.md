将来的にフルスタックエンジニアやセキュリティエンジニアを目標に勉強を進める中で、このタイミングでバックエンドに関することを勉強しだすのがちょうどいいのではないかと感じました。

まず、フロントエンド領域バックエンド領域の境目がどこか自分でもあいまいだったので領域の定義から調べました。
自分が普段使っているNext.js,Supabase,Vercelをまず調べてみました。

| スタック | 領域 | 自分の経験 |
| ---- | ---- | ---- | 
| Next.js | フロントエンド | Web関係のものを作るときは必ず使用している。 |
| Supabase | BaaS | 自分がよく使うDB。Next.js上に指定されたファイルを配置するだけで使用でき、RLSなどの設定はほぼ自力でやったことがない。(友人に頼る or AIに手順を提示してもらうなど) |
| Vercel | インフラ | 自分が作ったWebアプリ、サイトは全てこれでデプロイしている。ドメインの設定程度しか深く触っていない。 |

調べて分かったのがSupabaseがBaaS(Backend as a Service)でバックエンド領域であることだ。普段はSupabaseの指定通りにNext.js上にコーディングをしていたのでフロントエンドだけで完結するツールだと思っていた。
